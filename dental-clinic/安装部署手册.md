## 5. 配置 Gunicorn 生产环境

> ⚠️ **重要**：Flask 自带的开发服务器不适合生产使用，必须使用 Gunicorn。

### 5.1 创建 Gunicorn 配置文件

```bash
cat > /opt/dental-clinic/gunicorn_config.py << 'EOF'
# -*- coding: utf-8 -*-
# Gunicorn 配置文件 - 口腔诊所轻量化管理系统
# 适用于 2G 内存服务器

import multiprocessing

# 绑定地址和端口
bind = "0.0.0.0:5000"

# Worker 进程数（2G内存建议2-3个）
workers = 2

# Worker 类型（sync 适合轻量级应用）
worker_class = "sync"

# 每个 worker 最大请求数（防止内存泄漏）
max_requests = 1000
max_requests_jitter = 100

# 超时设置（秒）
timeout = 60
graceful_timeout = 30

# 日志
accesslog = "/opt/dental-clinic/logs/access.log"
errorlog = "/opt/dental-clinic/logs/error.log"
loglevel = "info"

# 进程命名
proc_name = "dental-clinic"

# 守护进程模式（配合 systemd 请用非守护模式）
daemon = False

# PID 文件
pidfile = "/opt/dental-clinic/dental-clinic.pid"
EOF

# 创建日志目录
mkdir -p /opt/dental-clinic/logs
```

### 5.2 创建上传目录

```bash
mkdir -p /opt/dental-clinic/static/uploads
```

### 5.3 测试 Gunicorn 是否能启动

```bash
# 临时测试，Ctrl+C 停止
cd /opt/dental-clinic
gunicorn -c gunicorn_config.py app:app
```

看到类似输出说明成功：
```
[INFO] Starting gunicorn...
[INFO] Listening at: http://0.0.0.0:5000
[INFO] Using worker: sync
[INFO] Booting worker with pid: xxxx
```

按 `Ctrl+C` 停止，继续下一步。

## 6. 配置 systemd 开机自启

### 6.1 写入服务文件

```bash
cat > /etc/systemd/system/dental-clinic.service << 'EOF'
[Unit]
Description=口腔诊所轻量化管理系统
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/dental-clinic
ExecStart=/usr/bin/gunicorn -c /opt/dental-clinic/gunicorn_config.py app:app
Restart=always
RestartSec=5
Environment=PYTHONUNBUFFERED=1

# 安全加固（可选）
NoNewPrivileges=true

[Install]
WantedBy=multi-user.target
EOF
```

### 6.2 启用并启动服务

```bash
# 重载 systemd 配置
systemctl daemon-reload

# 启动服务
systemctl start dental-clinic

# 查看状态
systemctl status dental-clinic

# 设置开机自启
systemctl enable dental-clinic

# 确认已启用
systemctl is-enabled dental-clinic
```

### 6.3 常用管理命令

```bash
# 启动
systemctl start dental-clinic

# 停止
systemctl stop dental-clinic

# 重启
systemctl restart dental-clinic

# 查看日志
journalctl -u dental-clinic -f

# 查看最近日志
journalctl -u dental-clinic -n 50 --no-pager
```

## 7. 配置阿里云安全组/防火墙

### 7.1 阿里云控制台 - 安全组

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com/)
2. 找到你的轻量应用服务器 → **安全** → **防火墙**
3. 添加规则：

| 方向 | 端口 | 协议 | 授权对象 | 说明 |
|------|------|------|----------|------|
| 入方向 | 5000 | TCP | 0.0.0.0/0 | 应用访问端口 |

> 💡 如果配置了 Nginx 反向代理（第8节），改为开放 80 和 443 端口

### 7.2 服务器内防火墙（如果开启了 firewalld/iptables）

```bash
# CentOS 7/8 用 firewalld
firewall-cmd --zone=public --add-port=5000/tcp --permanent
firewall-cmd --reload

# 或者直接关闭防火墙测试（不推荐生产环境长期关闭）
systemctl stop firewalld
```

### 7.3 验证端口是否可访问

在**本地 Windows** 执行：

```powershell
# PowerShell
Test-NetConnection -ComputerName 121.42.12.150 -Port 5000
```

## 8. 可选：配置 Nginx 反向代理 + HTTPS

> 💡 推荐配置，使用 80/443 标准端口，支持 HTTPS 加密访问

### 8.1 安装 Nginx

```bash
# CentOS
yum install -y nginx

# Ubuntu
apt install -y nginx
```

### 8.2 配置 Nginx

```bash
cat > /etc/nginx/conf.d/dental-clinic.conf << 'EOF'
server {
    listen 80;
    server_name 你的域名或IP;

    client_max_body_size 16M;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态文件直接由 Nginx 提供（提升性能）
    location /static/ {
        alias /opt/dental-clinic/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
EOF

# 测试配置
nginx -t

# 启动 Nginx
systemctl start nginx
systemctl enable nginx
```

### 8.3 配置后修改安全组

将安全组端口从 `5000` 改为 `80`（和 `443`），通过域名或 IP 直接访问即可。

### 8.4 配置 HTTPS（可选，需要 SSL 证书）

```bash
# 安装 Certbot
# CentOS
yum install -y certbot python3-certbot-nginx

# Ubuntu
apt install -y certbot python3-certbot-nginx

# 获取证书
certbot --nginx -d 你的域名

# 自动续期
certbot renew --dry-run
```

## 10. 日常运维

### 10.1 数据备份（重要‼️）

```bash
#!/bin/bash
# 备份脚本：保存到 /opt/backups/dental-clinic/

BACKUP_DIR="/opt/backups/dental-clinic"
mkdir -p $BACKUP_DIR

# 备份数据库
cp /opt/dental-clinic/dental.db $BACKUP_DIR/dental-$(date +%Y%m%d-%H%M).db

# 备份上传文件
tar -czf $BACKUP_DIR/uploads-$(date +%Y%m%d-%H%M).tar.gz /opt/dental-clinic/static/uploads/

# 保留最近30天的备份
find $BACKUP_DIR -name "*.db" -mtime +30 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

echo "备份完成: $(date)"
```

### 10.2 配置定时备份（crontab）

```bash
# 编辑定时任务
crontab -e

# 添加一行：每天凌晨3点自动备份
0 3 * * * /opt/dental-clinic/backup.sh >> /opt/dental-clinic/logs/backup.log 2>&1
```

保存备份脚本：

```bash
cat > /opt/dental-clinic/backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/opt/backups/dental-clinic"
mkdir -p $BACKUP_DIR
cp /opt/dental-clinic/dental.db $BACKUP_DIR/dental-$(date +%Y%m%d-%H%M).db
tar -czf $BACKUP_DIR/uploads-$(date +%Y%m%d-%H%M).tar.gz -C /opt/dental-clinic/static/ uploads/
find $BACKUP_DIR -name "*.db" -mtime +30 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete
echo "备份完成: $(date)"
EOF

chmod +x /opt/dental-clinic/backup.sh
```

## 11. 常见问题

### Q1: 启动报错 `ModuleNotFoundError: No module named 'flask'`

```bash
# 确认是否用 pip3 安装
pip3 list | grep flask

# 如果没有，重新安装
pip3 install -r /opt/dental-clinic/requirements.txt
```

### Q2: 访问页面 500 错误

```bash
# 查看错误日志
journalctl -u dental-clinic -n 50

# 常见原因：secret_key 在服务重启后 session 失效
# 解决：清除浏览器 Cookie，重新登录即可
```

### Q3: 端口 5000 连不上

```bash
# 检查服务是否运行
systemctl status dental-clinic

# 检查端口是否监听
netstat -tlnp | grep 5000

# 检查安全组/防火墙是否放行
# 阿里云控制台 → 安全 → 防火墙 → 添加 5000 端口
```

### Q4: 上传文件失败

```bash
# 检查上传目录权限
ls -la /opt/dental-clinic/static/uploads/

# 确保目录存在且有写入权限
mkdir -p /opt/dental-clinic/static/uploads
chmod 755 /opt/dental-clinic/static/uploads
```

### Q5: 内存占用过高

```bash
# 查看进程内存
ps aux | grep gunicorn

# 如果内存超预期，减少 worker 数
# 编辑 gunicorn_config.py，将 workers 改为 1
```

### Q6: 如何重置管理员密码？

```bash
cd /opt/dental-clinic
python3 << EOF
import sqlite3
from werkzeug.security import generate_password_hash
conn = sqlite3.connect('dental.db')
conn.execute("UPDATE users SET password_hash=? WHERE username='admin'", (generate_password_hash('新密码'),))
conn.commit()
conn.close()
print("密码已重置")
EOF
```
